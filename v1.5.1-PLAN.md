# v1.5.1 Release Plan: DCEL Office Suite Consolidation

This plan outlines the objectives, features, and fixes for the **v1.5.1** release of the DCEL Office Suite. The primary focus is on stabilizing recent features, refining the user experience (especially mobile and media capture), and addressing critical HR and reporting bugs.

## 1. Core Objectives
- **Consolidate Recent Features:** Bring Machine Log Deletions, Mobile Maintenance UI, and Admin Task Controls into a production-ready state.
- **UI/UX Polish & Branding:** Enhance the visual fidelity of the `CustomCamera` and introduce a high-fidelity "Invoice Reader" interface.
- **Critical Stability:** Fix high-priority bugs in the Employees module and Report Export functionality identified in `TODO-REVIEW.md`.

## 2. Feature Scope

### 2.1 Field Media & Document Extraction (Primary Design Focus)
- **WhatsApp-style Camera:** Refine `CustomCamera.tsx` with premium animations and responsive shutter logic (Tap-Photo/Hold-Video).
- **AI Invoice Reader:** Implement a document extraction interface for the Billing module, modeled after the successful CV parsing logic in `InterviewManager.tsx`.
- **Media Viewer:** Stabilize the `MediaViewer.tsx` for seamless playback of captured field media.

### 2.2 Operational Enhancements
- **Machine Log Deletions:** Implement secure deletion logic for maintenance records, gated by `operations.canDeleteLogs`.
- **Mobile Maintenance UI:** Optimize tabbed navigation and summary cards for field use on small screens.
- **Admin "God Mode":** Ensure users with Management access can override, edit, or delete any task reminders.

### 2.3 Critical Bug Fixes (from TODO-REVIEW.md)
- **Employees Module:** Fix the Edit and View buttons to open functional modals instead of placeholders.
- **Reports Module:** Implement functional CSV/Excel export logic for all report types.
- **Settings & Variables:** Ensure changes to system variables and settings are correctly persisted to the store.

## 3. Implementation Schedule

### Phase 1: Stability Pass (Current)
- Fix Employees Edit/View flow.
- Implement functional exports for the Reports page.
- Ensure persistence for Variables and Settings.

### Phase 2: Feature Polish
- Apply branding and animations to `CustomCamera`.
- Integrate the Invoice Reader into the Billing workflow.
- Implement Log Deletion confirmation dialogs.

### Phase 3: Final Integration & Review
- Run `checklist.py` to verify all modules.
- Perform a cross-browser/mobile responsive audit.
- Finalize documentation.

## 4. Success Criteria
- Version incremented to 1.5.1 in `package.json`.
- Zero placeholder alerts remaining in the core HR and Financial workflows.
- Successful end-to-end media capture and invoice parsing (mock AI).
