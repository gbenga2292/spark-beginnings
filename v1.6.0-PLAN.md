# v1.6.0 Release Plan: Client 360 Refinements & Global Integration

This plan outlines the objectives, feature scope, and success criteria for the **v1.6.0** release of the DCEL Office Suite. The primary focus of this release is on consolidating Client 360 analytics, task tagging workflows, ledger filtering capabilities, and bumping application version settings across all desktop, web, and mobile configurations.

## 1. Core Objectives
- **Client 360 UX & Structure Refinement**: Stabilize and rebuild the Client 360 tabbed workspace, ensuring tasks, operations, and financials are properly aggregated.
- **Operations & Navigation Enhancements**: Restructure the Operations views into readable layouts, replace redundant icons with standard dropdown ellipsis, and remove obsolete metrics.
- **Global Header Search**: Integrate Client 360 direct search within the main header layout for rapid navigation.
- **Ledger Financial Precision**: Introduce live filtered totals within the Ledger views to simplify account audits.
- **Cross-Platform Version Alignment**: Align the application version to `1.6.0` across the web environment, native Capacitor/Android configurations, and the Electron desktop wrapper.

## 2. Feature Scope

### 2.1 Client 360 & Operations Optimization
- **Tasks Tab Consolidation**: Repair Client 360 workspace tab structure to accurately roll up site tasks and subtasks.
- **Grid Layouts**: Restructure the Operations dashboard tab into a two-column responsive grid layout to maximize screen real estate.
- **Financial Clean-up**: Remove obsolete "Estimated Profit" and "Actual Cash Profit" cards to streamline billing and reporting dashboards.
- **Ellipsis Navigation**: Replace old view icons with a standard MoreHorizontal ellipsis dropdown for cleaner row controls.

### 2.2 Global Header Search & Ledger Auditing
- **Quick-Search Client 360**: Add an autocomplete search trigger in the application header for instant access to Client 360 profiles.
- **Ledger Totals**: Display live, filtered summary totals at the top of the Ledger table to support real-time financial tracking.

### 2.3 Task Tagging & Workflows
- **Site/Client Task Association**: Enable tagging of master tasks and subtasks to specific clients and construction sites.
- **Standardized Status Control**: Apply the new "To Start" status nomenclature across all task register cards, dashboards, and reporting modules.

## 3. Version Bump Scope
To deploy version 1.6.0, the version string must be incremented in the following locations:
- `package.json` (core node app metadata)
- `android/app/build.gradle` (native Android version name and incremented build version code)
- `src/pages/Settings.tsx` (system settings panel)
- `src/components/layout/Header.tsx` (app header indicator)
- `src/components/layout/Sidebar.tsx` (app update checker and footer version)
- `README.md` (project documentation and change logs)
- `android-update-setup.md` (apk private update documentation examples)

## 4. Success Criteria
- Native Android build config version incremented to code `15` and name `1.6.0`.
- System settings and update checks successfully recognize `1.6.0` as the current version.
- Package lock updated and verified.
- Build verifies with zero TypeScript compiler errors.
